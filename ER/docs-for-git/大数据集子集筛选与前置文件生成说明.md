# 大数据集子集筛选与前置文件生成说明

更新日期：2026-06-17

## 1. 处理目标

为降低正式实验训练成本，同时避免子集规模与 Eedi-sub 差距过大，本次重新构造两个较大数据集的子集：

- `assist2009-sub`
- `XES3G5M-sub-small`

参考目标是 Eedi-sub 当前 ER 图规模：

| Dataset | Total triples |
| --- | ---: |
| Eedi-sub | 999,980 |

因此新子集目标范围设为 Eedi-sub 的约 ±20%：

```text
800,000 ~ 1,200,000 triples
```

## 2. 筛选原则

本次没有只按学生作答长度筛选，而是采用学生、题目、知识点三方约束：

1. 先排除作答序列过短或过长的学生。
2. 再做知识点分层题目采样，优先保证知识点覆盖。
3. 再按题目作答热度补齐目标题目数。
4. 重新过滤题目采样后有效作答不足的学生。
5. 保持知识点 ID 原始宽度不压缩，便于 KT/DKT 导出和跨数据集解释。

题目目标数使用固定随机种子 `2026` 从范围内抽取，并跳过整十、整百这类过于规整的数值：

| Dataset | Question target range | Selected target |
| --- | ---: | ---: |
| ASSISTments 2009 | 约 1,900 ~ 2,200 | 1,987 |
| XES3G5M-sub | 约 1,800 ~ 1,950 | 1,928 |

## 4. 筛选参数

### assist2009-sub

```text
source: ER/KG4ER/data/assist2009/prepared_for_kt
output: ER/KG4ER/data/assist2009-sub
student sequence length: 10 ~ 200
target questions: 1987
min question interactions: 1
min questions per concept: 3
split ratio: 0.75
subset seed: 2026
concept id policy: preserve source Q width
```

说明：ASSISTments 2009 的题目-知识点分布长尾明显。如果设置较高题目交互阈值，会导致大量低频知识点被过滤。因此最终将 `min_question_interactions` 设为 `1`，并通过知识点分层采样保证结构覆盖。

### XES3G5M-sub-small

```text
source: ER/KG4ER/data/XES3G5M-sub/prepared_for_kt
output: ER/KG4ER/data/XES3G5M-sub-small
student sequence length: 10 ~ 400
target questions: 1928
min question interactions: 20
min questions per concept: 3
split ratio: 0.75
subset seed: 2026
concept id policy: preserve source Q width
```

说明：XES3G5M-sub 如果按 Eedi-sub 的 `10~200` 过滤会只剩约 294 名学生，规模过小。因此上限调整为 `400`，同时保留全部 408 个活跃知识点。

## 6. 生成命令

### assist2009-sub

```powershell
python ER\KG4ER\data\build_structured_er_subset.py `
  --source-dir ER\KG4ER\data\assist2009\prepared_for_kt `
  --output-dir ER\KG4ER\data\assist2009-sub\processed `
  --dataset-name assist2009-sub `
  --target-questions 1987 `
  --min-student-interactions 10 `
  --max-student-interactions 200 `
  --min-question-interactions 1 `
  --min-questions-per-concept 3 `
  --split-ratio 0.75 `
  --seed 2026
```

### XES3G5M-sub-small

```powershell
python ER\KG4ER\data\build_structured_er_subset.py `
  --source-dir ER\KG4ER\data\XES3G5M-sub\prepared_for_kt `
  --output-dir ER\KG4ER\data\XES3G5M-sub-small\processed `
  --dataset-name XES3G5M-sub-small `
  --target-questions 1928 `
  --min-student-interactions 10 `
  --max-student-interactions 400 `
  --min-question-interactions 20 `
  --min-questions-per-concept 3 `
  --split-ratio 0.75 `
  --seed 2026
```
