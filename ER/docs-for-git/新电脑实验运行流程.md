# 新电脑实验运行流程

本文档用于在一台新电脑上拉取 GitHub 代码、拷贝本地数据，并运行 ER/KG4ER 实验。当前 GitHub 仓库只保存代码和轻量说明文档，不保存数据集、PDF、模型权重、运行日志和 `ER/docs/` 本地文档。

## 1. 需要安装的软件

建议使用 Windows + Anaconda/Miniconda 环境。

必须安装：

- Git：用于拉取 GitHub 仓库。
- Anaconda 或 Miniconda：用于创建 Python 环境。
- Python 3.9 或 3.10：推荐使用 3.9。
- PyTorch：优先安装 GPU 版本。
- 其他 Python 库：由 `ER/KG4ER/requirements.txt` 安装。

如果使用 GPU，还需要：

- NVIDIA 显卡驱动。
- 与 PyTorch 匹配的 CUDA 运行环境。一般不需要单独安装完整 CUDA Toolkit，优先按 PyTorch 官网给出的命令安装。

## 2. 拉取代码

在新电脑选择一个工作目录，例如：

```powershell
cd D:\Research
git clone https://github.com/Tangfgjk/KG4ER.git
cd KG4ER
```

仓库主要结构：

```text
ER/
  KG4ER/
    codes/          # 训练、测试、一键实验、baseline、消融实验
    data/           # 数据处理脚本，不含真实数据
    tests/          # 测试代码
    requirements.txt
  docs-for-git/     # 可上传 GitHub 的轻量文档
```

## 3. 创建环境并安装依赖

创建并进入环境：

```powershell
conda create -n kg4er python=3.9 -y
conda activate kg4er
```

安装 PyTorch。GPU 版本请按当前电脑 CUDA 情况选择，例如 CUDA 12.1 可参考：

```powershell
pip install torch --index-url https://download.pytorch.org/whl/cu121
```

如果只用 CPU：

```powershell
pip install torch
```

然后安装项目依赖：

```powershell
cd ER\KG4ER
pip install -r requirements.txt
```

检查 CUDA 是否可用：

```powershell
python -c "import torch; print(torch.cuda.is_available()); print(torch.cuda.get_device_name(0) if torch.cuda.is_available() else 'CPU')"
```

如果输出 `True`，一键实验命令中的 `--cuda auto` 会优先使用 GPU。

## 4. 拷贝数据

GitHub 仓库不包含数据。需要把已经生成好的前置文件单独拷贝到新电脑。

数据目录应放在：

```text
ER/KG4ER/data/Eedi/
ER/KG4ER/data/algebra2005/prepared_for_kt/
ER/KG4ER/data/assist2009/prepared_for_kt/
ER/KG4ER/data/assist2009-sub/prepared_for_kt/
ER/KG4ER/data/statics2011/prepared_for_kt/
ER/KG4ER/data/XES3G5M-sub/prepared_for_kt/
ER/KG4ER/data/XES3G5M-sub-small/prepared_for_kt/
```

每个实验数据目录至少需要包含：

```text
Q.txt
stu2know_mastery.json
stu2know_seq.json
stu2know_forget.json
stu2ex_forget.json
stu2ex_recommend.json
triples.txt
test_triples.txt
entities.dict
relations.dict
```

如果要运行传统基线 `EB-CF`、`SB-CF`，建议同时保留：

```text
sequence_interactions.csv
```

注意：

- 如果这些前置文件已经生成好，新电脑只需要拷贝数据，不需要拷贝 `EL/`、`KT/` 或 `ER/pykt-toolkit-main/`。
- 当前 ER 训练和测试阶段主要使用 `ER/KG4ER/data/` 下的前置文件。
- `relations.dict` 应保持 304 个关系。

## 5. 单随机种子运行实验

所有训练命令都在下面目录执行：

```powershell
cd ER\KG4ER\codes
```

先建议只跑一个随机种子，例如 `2024`：

```powershell
python run_dataset_experiments.py `
  --dataset algebra2005 `
  --seeds 2024 `
  --cuda auto
```

可选数据集名称：

```text
Eedi
algebra2005
assist2009
assist2009-sub
statics2011
XES3G5M-sub
XES3G5M-sub-small
```

其中 `assist2009-sub` 和 `XES3G5M-sub-small` 是为了降低训练成本重新筛选出的子集，规模更接近 Eedi-sub，适合正式多种子实验优先使用。

例如运行 Eedi：

```powershell
python run_dataset_experiments.py `
  --dataset Eedi `
  --seeds 2024 `
  --cuda auto
```

默认会运行该数据集下的全部实验，包括：

- 主模型：`ConvE_full`
- 消融实验：`ConvE_no_seq`、`ConvE_no_forgetting`、`ConvE_no_mastery`
- KGE 对比模型：`TransE`、`TransE-adv`、`RotatE`、`DistMult`、`ComplEx`
- 传统 baseline：`EB-CF`、`SB-CF`、`CBF`、`KCP-ER`

运行结果会保存到：

```text
ER/KG4ER/runs/<dataset>/<dataset>_full_<时间戳>/
```

例如：

```text
ER/KG4ER/runs/algebra2005/algebra2005_full_20260616-153000/
```

## 6. 正式实验随机种子

正式实验建议使用 5 个随机种子：

```text
2024, 2025, 2026, 2027, 2028
```

正式运行命令：

```powershell
python run_dataset_experiments.py `
  --dataset algebra2005 `
  --seeds 2024,2025,2026,2027,2028 `
  --cuda auto
```

这样可以统计 `mean ± std`。

如果多台电脑分开跑，建议每台电脑固定一个数据集；如果只想临时验证流程，就先只跑 `--seeds 2024`。

## 7. 按图1和图2分组运行

如果不想一次运行所有实验，可以按论文表格中的两组模型拆开跑。建议两组命令使用同一个 `--run-id`，这样图1和图2的结果会保存到同一个实验目录，后续汇总更方便。

下面示例以 `algebra2005` 和单个随机种子 `2024` 为例。正式实验时可以把 `--seeds 2024` 改成 `--seeds 2024,2025,2026,2027,2028`。

图1模型包括：

```text
ConvE_full, ConvE_no_seq, ConvE_no_forgetting, ConvE_no_mastery, TransE, TransE-adv
```

首次运行图1模型：

```powershell
python run_dataset_experiments.py `
  --dataset algebra2005 `
  --run-id algebra2005_grouped_2024 `
  --seeds 2024 `
  --models ConvE_full,ConvE_no_seq,ConvE_no_forgetting,ConvE_no_mastery,TransE,TransE-adv `
  --cuda auto
```

图1模型续跑：

```powershell
python run_dataset_experiments.py `
  --dataset algebra2005 `
  --run-id algebra2005_grouped_2024 `
  --seeds 2024 `
  --models ConvE_full,ConvE_no_seq,ConvE_no_forgetting,ConvE_no_mastery,TransE,TransE-adv `
  --resume `
  --cuda auto
```

图2模型包括：

```text
RotatE, DistMult, ComplEx, EB-CF, SB-CF, CBF, KCP-ER
```

首次运行图2模型：

```powershell
python run_dataset_experiments.py `
  --dataset algebra2005 `
  --run-id algebra2005_grouped_2024 `
  --seeds 2024 `
  --models RotatE,DistMult,ComplEx,EB-CF,SB-CF,CBF,KCP-ER `
  --cuda auto
```

图2模型续跑：

```powershell
python run_dataset_experiments.py `
  --dataset algebra2005 `
  --run-id algebra2005_grouped_2024 `
  --seeds 2024 `
  --models RotatE,DistMult,ComplEx,EB-CF,SB-CF,CBF,KCP-ER `
  --resume `
  --cuda auto
```

如果在不同电脑上分别跑图1和图2，也可以使用不同的 `--run-id`，例如 `algebra2005_fig1_2024` 和 `algebra2005_fig2_2024`。这种做法不会影响单个模型结果，但后续需要手动或额外脚本把两个运行目录下的 `summaries/` 合并到同一张论文表格。

## 8. 续跑实验

如果实验中断，可以使用 `--resume` 续跑。

只续跑上次的一个随机种子，例如之前只跑了 `seed2024`：

```powershell
python run_dataset_experiments.py `
  --dataset algebra2005 `
  --seeds 2024 `
  --resume `
  --cuda auto
```

这个命令会：

- 自动找到 `ER/KG4ER/runs/algebra2005/` 下最新的一次运行目录。
- 只检查并续跑 `--seeds 2024` 指定的随机种子。
- 已完成的实验会跳过。
- 未完成的 ConvE 实验会优先读取该 seed 目录下的 `last.pt` 继续训练。
- 未完成的 KGE 实验会优先读取该 seed 目录下的 `checkpoint` 继续训练。

重要说明：

- `--resume` 能识别最新 run 目录，也能根据目录结构识别 `seed2024` 是否已经完成。
- 但它不会自动猜测“上次你只跑了哪个随机种子”。
- 因此，如果你只想续跑上次的一个随机种子，必须继续显式写 `--seeds 2024`。
- 如果续跑时不写 `--seeds`，脚本会使用默认值 `2024,2025,2026,2027,2028`，也就是会检查 5 个随机种子。

如果不想自动使用最新 run 目录，也可以指定某一次运行：

```powershell
python run_dataset_experiments.py `
  --dataset algebra2005 `
  --run-id algebra2005_full_20260616-153000 `
  --seeds 2024 `
  --resume `
  --cuda auto
```

## 9. 常用参数

默认参数已经可以直接跑实验。常用参数如下：

```text
--dataset              数据集名称
--seeds                随机种子，单个如 2024，多个如 2024,2025,2026,2027,2028
--cuda                 auto / true / false，推荐 auto
--resume               续跑最新一次该数据集实验
--run-id               指定某一次 run 目录
--epochs               ConvE 训练轮数，默认 10
--conve-input-drop     ConvE input dropout，默认 0.2
--conve-hidden-drop    ConvE hidden dropout，默认 0.2
--conve-feat-drop      ConvE feature map dropout，默认 0.3
--conve-include-test-triples  可选开关，显式允许 ConvE 训练时额外使用 test_triples.txt
--kge-max-steps        TransE/RotatE/DistMult/ComplEx 最大训练 step，默认 10000
--models               指定只跑部分模型，默认 all
```

当前一键实验中的 ConvE 默认采用与原论文/旧代码更一致的训练协议：

- 训练轮数默认 `10`。
- dropout 默认为 `input_drop=0.2`、`hidden_drop=0.2`、`feat_drop=0.3`。
- 默认只使用 `triples.txt` 训练，不额外把 `test_triples.txt` 加入训练。
- 如果需要复现实验排查中的旧工程做法，可以显式添加 `--conve-include-test-triples`。

只跑主模型示例：

```powershell
python run_dataset_experiments.py `
  --dataset algebra2005 `
  --seeds 2024 `
  --models ConvE_full `
  --cuda auto
```

只跑某几个模型示例：

```powershell
python run_dataset_experiments.py `
  --dataset algebra2005 `
  --seeds 2024 `
  --models ConvE_full,TransE,RotatE `
  --cuda auto
```

## 10. 输出结果

每个模型和随机种子会有独立目录，例如：

```text
ER/KG4ER/runs/algebra2005/algebra2005_full_<时间戳>/ConvE_full/seed2024/
```

常见输出包括：

```text
train.log                 # 训练日志
test.log                  # 测试/打分日志
eval.log                  # 指标计算日志
timing.json               # 时间统计
best.pt                   # ConvE 最佳模型
last.pt                   # ConvE 最新模型
checkpoint                # KGE 断点目录/文件
eval/metrics.json         # 指标 JSON
eval/metrics.csv          # 指标 CSV
explanations/             # ConvE 解释卡片
```

汇总结果位于：

```text
ER/KG4ER/runs/<dataset>/<run-id>/summaries/
```

其中常用文件包括：

```text
dataset_summary.csv
dataset_summary.md
aggregate_metrics.csv
```

## 11. 推荐的新电脑最小流程

第一次在新电脑上建议按下面流程：

```powershell
git clone https://github.com/Tangfgjk/KG4ER.git
cd KG4ER

conda create -n kg4er python=3.9 -y
conda activate kg4er

cd ER\KG4ER
pip install -r requirements.txt

# 手动拷贝数据到 ER/KG4ER/data/

cd codes
python run_dataset_experiments.py `
  --dataset Eedi `
  --seeds 2024 `
  --cuda auto
```

如果中断后续跑：

```powershell
python run_dataset_experiments.py `
  --dataset Eedi `
  --seeds 2024 `
  --resume `
  --cuda auto
```
